Release Notes for Quaker v2.0.2

14 June 2011

 

Quaker v2.0.2 is an update for the first release of Quaker HD v2 for webOS 3.0 based on the Enyo framework.  The following functionality is still being developed/updated but is not likely to make it into the first public release:

 

1)      Filtering Capabilities - allow the user to select filter criteria such as: distance from a location (city, lat/lon, current pos, etc..), magnitude, age of event and apply this filter to notifications.

 

2)      Background notifications – Dashboard functionality is working, but currently Quaker only runs while “open”. Need to set up a periodic check for new quakes and notify via dashboard in the background. This might be something I defer for an update after release. It’s not essential to the functionality but is highly desirable.

 

Major Changes from Pervious Release:

 

1)      Many UI enhancements (icons, color coded magnitudes for pins and list, list header less cluttered, webView loading improved)

2)      Start up spinner to show activity while app initializes

3)      Improved notification handling

4)      Detection of no data connection on start up

5)      Detection of Location Services not available.

6)      Addition of “auto select” and “auto locate” options in preferences.
